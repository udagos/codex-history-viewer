# Plan: Folder View, Sorting, and Folder Pin/Tag Support

## Objective
1. **Show by Folder**: Add a new view mode to the History tree to group sessions by folder.
2. **Folder Sorting**: Allow sorting folders by Alphabetical (A-Z), Created Date (oldest session), or Modified Date (newest session).
3. **Folder Pins & Tags**: Pinning or tagging a folder should apply to the folder itself. Sessions within that folder inherit these properties (e.g., filtering by a tag shows all sessions in the tagged folder).
4. **Categorization**: Distinguish between folders and individually marked sessions in the Pinned view (and search/filter views) using categories/groups.

---

## File Modifications

### 1. `src/sessions/sessionTypes.ts`
- **HistoryIndex**: Add `byFolder: Map<string, SessionSummary[]>` to store sessions grouped by their workspace folder (`cwd`).

### 2. `src/services/historyService.ts`
- **buildIndex**: Initialize `byFolder`. Extract `session.meta.cwd` and populate the map. Sort the sessions within each folder by `localDate` and `timeLabel` (descending).

### 3. `src/tree/treeNodes.ts`
- **FolderNode**: Define `FolderNode` with `kind = "folder"`, `cwd`, `cwdShort`, and `pinned`.
- **Group Nodes**: Define `PinnedFolderGroupNode` and `PinnedSessionGroupNode` to categorize items in the Pinned view.
- **TreeNode**: Add the new nodes to the union type.
- **Context Values**: Update `toTreeItemContextValue` for the new nodes to enable specific context menu commands.

### 4. `src/tree/historyTree.ts`
- **HistoryViewMode**: Update to `"date" | "latest" | "folder"`.
- **State**: Add `folderSortMode: "alphabetical" | "created" | "modified"` (loaded from `workspaceState`).
- **matchesTags**: Update logic so a session matches if its own `fsPath` is tagged OR if its `meta.cwd` is tagged.
- **getChildren**:
  - In `folder` mode, yield `FolderNode`s.
  - Implement folder sorting:
    - *Alphabetical*: by `cwdShort`.
    - *Created*: ascending by the minimum `startedAtIso` (or `localDate`) of sessions in the folder.
    - *Modified*: descending by the maximum `lastActivityAtIso` (or `localDate`) of sessions in the folder.

### 5. `src/tree/pinnedTree.ts`
- **Categorization**: 
  - Update `getChildren` at the root level to return `PinnedFolderGroupNode` and `PinnedSessionGroupNode` if items exist.
  - Under `PinnedFolderGroupNode`, return pinned `FolderNode`s (which can be expanded to show their sessions).
  - Under `PinnedSessionGroupNode`, return pinned `SessionNode`s.
- **matchesTags**: Similar to `historyTree.ts`, ensure folder tags are respected.

### 6. `src/extension.ts`
- **Folder Action Resolution**: 
  - Update `resolveTargets` / `resolveAnnotationTargets` so that when a `FolderNode` is selected, it returns the folder's `cwd` instead of expanding to its child sessions. This allows bulk tagging/pinning the folder itself via `PinStore` and `SessionAnnotationStore`.
- **Sorting Commands**:
  - Register `codexHistoryViewer.sortFolderByAlphabetical`
  - Register `codexHistoryViewer.sortFolderByCreated`
  - Register `codexHistoryViewer.sortFolderByModified`
  - These commands update `workspaceState` and refresh the history provider.
- **View Mode Commands**:
  - Register `codexHistoryViewer.showHistoryFolderView` to switch to folder mode.

### 7. `package.json`
- **Commands**: Add the new view mode and sorting commands.
- **Menus**: 
  - `view/title`: Add the folder sorting commands as a submenu or inline actions when `historyViewMode == 'folder'`. Update view mode cycle logic.
  - `view/item/context`: Ensure pin/unpin/tag commands appear for `FolderNode` context (`codexHistoryViewer.folder`).

### 8. Localization (`package.nls.json` and `package.nls.ja.json`)
- Add translations for:
  - "Show by Folder" / "フォルダー別で表示"
  - Sort commands: "Sort Folders: Alphabetical", "Sort Folders: Created", "Sort Folders: Modified".
  - Group node labels: "Pinned Folders", "Pinned Sessions".
